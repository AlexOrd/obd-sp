Простий шаблон для швидкого старту
Супер простий шаблон для Telegram-бота на Python з бібліотекою aiogram.
Базовий режим — кнопки, текстові відповіді, привітання.
Режим ШІ — повноцінний діалог з OpenAI Assistants API. Новинка 🎉
Всі налаштування через GitHub Secrets — ніяких ключів у коді!
Запускаємо все у GitHub Codespaces — нічого встановлювати на ПК.
🚀
Поїхали! GitHub Codespaces
Найпростіший спосіб запустити проект
☁️ Що таке Codespaces?
Повноцінне середовище розробки прямо у браузері. Сервер у хмарі, VS Code онлайн, термінал — все готово за хвилину.
✅ Чому це зручно?
Не потрібно нічого встановлювати. Працює з будь-якого пристрою. Secrets зберігаються в GitHub — безпечно!
ℹ️ GitHub дає 60 годин безкоштовно на місяць для Codespaces — цього більш ніж достатньо для навчання!
🍴
Крок 1 — Fork репозиторію
Робимо власну копію шаблону
1
Відкрийте репозиторій шаблону на GitHub: github.com/AlexOrd/vtfk_tg_bot
2
Натисніть кнопку "Fork" у правому верхньому куті сторінки.
Або скористайтесь кнопкою "Use this template" — обидва варіанти дають однаковий результат.
3
У вашому GitHub-акаунті з'явиться власна копія репозиторію — саме з нею ми будемо працювати далі.
☁️
Крок 2 — Запускаємо Codespace
Хмарний ПК готовий за ~1 хвилину
1
На сторінці вашого форку натисніть зелену кнопку "< > Code".
2
Перейдіть на вкладку "Codespaces".
3
Натисніть "Create codespace on main".
GitHub запустить хмарний сервер і відкриє VS Code прямо в браузері.
✅ Зачекайте ~30–60 секунд поки середовище повністю завантажиться.
🔐
Крок 3 — Налаштовуємо Secrets
Ключі ніколи не повинні потрапляти в код!
⚠️ Важливо: Токени та API-ключі зберігаємо виключно в GitHub Secrets, а не прямо у файлах проекту.
✓
BOT_TOKEN — токен вашого бота від @BotFather в Telegram. Обов'язково
○
OPENAI_API_KEY — API-ключ від OpenAI (platform.openai.com/api-keys). Для режиму ШІ.
○
ASSISTANT_ID — ID вашого OpenAI-асистента (починається з asst_...). Для режиму ШІ.
⚙️
Як додати Secret у GitHub
Settings → Secrets and variables → Codespaces
1
Перейдіть до Settings вашого репозиторію (вкладка вгорі).
2
У бічному меню знайдіть "Secrets and variables" → "Codespaces".
3
Натисніть "New repository secret".
4
Введіть Name (наприклад, BOT_TOKEN) та Value (ваш токен). Повторіть для кожного ключа.
💡 Після збереження Secret перезапустіть Codespace, щоб нові змінні середовища підтягнулись.
🖥️
Для локального запуску
Замість Secrets використовуємо змінні середовища

Якщо запускаєте бота не через Codespaces, а локально на своєму ПК, встановіть змінні середовища перед запуском:

bash / zsh
export BOT_TOKEN="ваш-токен-бота"
export OPENAI_API_KEY="ваш-openai-ключ"   # опціонально
export ASSISTANT_ID="ваш-assistant-id"    # опціонально
ℹ️ Ці команди діють лише до закриття поточного термінального сеансу. Для постійного зберігання додайте їх у ~/.bashrc або ~/.zshrc.
📦
Крок 4 — Встановлення залежностей
Одна команда — і всі бібліотеки на місці

Перед першим запуском переконайтеся, що всі потрібні бібліотеки встановлено. Введіть у термінал:

terminal
pip install -r requirements.txt
✅ Зазвичай Codespaces робить це автоматично при запуску. Але краще перевірити вручну — це займає лише кілька секунд!
📄 Список залежностей знаходиться у файлі requirements.txt у корені репозиторію.
▶️
Крок 5 — Запускаємо бота!
Момент істини 🎉

Тепер все готово! У терміналі внизу введіть команду:

terminal

🎉 Якщо побачите рядок INFO:aiogram.dispatcher:Bot is starting... — ваш бот живий і працює!

Що побачите в терміналі

OpenAI Assistants API

Потужний інструмент для ШІ-асистентів із пам'яттю

Контекст розмови Threads API gpt-4o-mini
⚡
Що вміє Assistants API?
Чим відрізняється від звичайного ChatGPT
Запам'ятовує контекст розмови — на відміну від простих запитів до ChatGPT.
Використовує треди (Threads) для збереження всієї історії діалогу.
Має попередньо налаштовані інструкції та поведінку — ви "навчаєте" асистента заздалегідь.
Кожен користувач бота отримує свій окремий thread — ніяких змішань розмов.
🛠️
Як створити OpenAI Assistant
platform.openai.com/assistants
1
Перейдіть на platform.openai.com/assistants та натисніть "Create".
2
Налаштуйте асистента: Name (наприклад, "Знавець ВТФК"), Instructions — опишіть як він має поводитися.
3
Оберіть модель. Рекомендуємо gpt-4o-mini — швидка й економна.
4
Збережіть та скопіюйте Assistant ID (починається з asst_...). Додайте його в GitHub Secrets як ASSISTANT_ID.
🧵
Як зберігається контекст
Threads — нитки розмов
Перше повідомлення
→
Новий Thread
Наступне повідомлення
→
Той самий Thread
🧵 Кожен користувач отримує свій thread. Асистент бачить всю попередню історію розмови. Thread зберігається в пам'яті бота поки він працює.
💬
Приклад роботи з контекстом
Асистент пам'ятає попередні повідомлення ✨
telegram chat
Користувач: Привіт! Як справи?
Асистент  : Привіт! У мене все чудово. Чим можу допомогти?

Користувач: Що ти сказав у попередньому повідомленні?
Асистент  : Я сказав, що у мене все чудово
             і запитав, чим можу допомогти.  # ← пам'ятає!
✨ Це справжній контекстний діалог — не просто відповідь на одне повідомлення!

Крок 2 — Оновлюємо bot.py
Три маленьких додавання
bot.py
# 1. Читаємо тексти в змінні
MSG_HELP = read_message('msg_help')
BTN_HELP = read_message('btn_help')

# 2. Додаємо кнопку в клавіатуру
main_keyboard = ReplyKeyboardMarkup(
    keyboard=[
        # ... існуючі кнопки ...
        [KeyboardButton(text=BTN_HELP)]  # ← нова кнопка!
    ],
    resize_keyboard=True
)

# 3. Обробник натискання кнопки
@dp.message(lambda m: m.text == BTN_HELP)
async def handle_help_button(message: types.Message):
    await message.answer(MSG_HELP)
